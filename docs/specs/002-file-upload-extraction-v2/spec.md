# 功能规格说明书 — 文件上传与事件抽取改造 v2

- **阶段编号**: 002
- **功能名称**: file-upload-extraction-v2
- **版本**: 2.0
- **状态**: 草稿

---

## 1. 概述

### 1.1 目标

改造文件上传和 AI 事件抽取流程，实现：
- 文件生命周期追踪（上传→切片→抽取→完成）
- 文本智能切片（带重叠，防语义断裂）
- LangChain 结构化事件抽取（自动解析 + 人物关联）
- 前端文件管理表格 + 切片详情页

### 1.2 范围

#### 包含功能

| 模块 | 功能 | 优先级 |
|------|------|--------|
| 文件上传 | 磁盘持久化 + 上传记录追踪 + 状态管理 | P0 |
| 文本切片 | 异步切片 + 重叠策略 + 边界信息保存 | P0 |
| 上传记录管理 | 文件列表查询、详情、删除（含物理删除） | P0 |
| 人员搜索 | 模糊搜索人物姓名 | P1 |
| AI 事件抽取 | LangChain 结构化输出 + persons 字段 + 自动人物关联 | P0 |
| 切片列表页 | 切片只读展示 + 全文查看 | P1 |
| 文件管理页 | 文件表格 + 状态标签 + 操作按钮（抽取/切片/删除） | P0 |

#### 不包含功能

- 批量文件上传（第二阶段）
- OCR 图片文字识别（第三阶段）
- 抽取结果人工审核流程（后续补充）
- 文件下载功能（后续补充）
- 更多文件格式支持（如 docx、epub）

## 2. 功能规格

### 2.1 文件上传

**功能描述**: 用户选择人物后上传 TXT/PDF 文件，系统保存到磁盘并记录到数据库。

**支持格式**: `.txt`、`.pdf`

**文件大小限制**: 最大 10MB

**存储规则**:
- 存储路径：`uploads/{uuid}.{ext}`
- 文件名：`uuid4().hex` + 原始扩展名
- 示例：`uploads/a1b2c3d4e5f67890abcdef123456789.pdf`

**交互流程**:
1. 用户在搜索框中选择关联人物
2. 点击上传按钮或拖拽文件到上传区域
3. 系统保存文件到磁盘 → 创建 `uploaded_files` 记录（status=uploaded）
4. 后台异步启动文本切片任务
5. 页面实时展示文件状态变更

**状态流转**:

```
┌──────────┐    切片开始    ┌──────────┐    切片完成    ┌──────────┐
│ uploaded │ ────────────→ │ chunking │ ────────────→ │ chunked  │
└──────────┘               └──────────┘               └──────────┘
     │                          │                          │
     │ 切片失败                  │ 切片失败                  │ 抽取开始
     ↓                          ↓                          ↓
┌──────────┐              ┌──────────┐               ┌────────────┐
│  error   │              │  error   │               │ extracting │
└──────────┘              └──────────┘               └────────────┘
                                                          │
                                                    抽取完成││抽取失败
                                                          ↓↓
                                                     ┌──────────┐  ┌──────────┐
                                                     │completed │  │  error   │
                                                     └──────────┘  └──────────┘
```

**错误处理**:
- 文件类型校验失败 → 400 错误提示
- 文件大小超限 → 413 错误提示
- 切片/抽取过程异常 → status 变为 error，error_message 记录异常信息
- 用户可点击"切片"按钮手动重试（status=error 的记录）

### 2.2 文本切片

**功能描述**: 文件上传后自动将文本切分为有重叠的片段，避免机械切割导致语义丢失。

**切片参数**:

| 参数 | 值 | 说明 |
|------|-----|------|
| chunk_size | 500 字符 | 每个切片最大字符数 |
| chunk_overlap | 50 字符 | 相邻切片重叠字符数 |
| separators | `["\n\n", "\n", "。", "！", "？", "；", ".", "!", "?", ";", " "]` | 递归分隔符优先级 |

**实现方式**: `langchain_text_splitters.RecursiveCharacterTextSplitter`

**切片存储**: 每个切片存入 `biography_text` 表，包含：
- `file_id` — 来源文件 ID
- `file_path` — 来源文件路径
- `chunk_index` — 切片序号（从 0 起）
- `raw_text` — 切片文本内容
- `person_id` — 关联人物 ID

**边界信息**: 相邻切片的 `chunk_overlap` 确保关键信息不会被切断。例如 "赤壁之战发生于建安十三年" 不会被切分为 "赤壁之战发生于建安" + "十三年"。

### 2.3 事件抽取

**功能描述**: 调用 LangChain + LLM 从文件的所有切片中提取结构化事件，自动关联/创建人物。

**抽取流程**:

```
遍历文件所有切片
  → 每个切片调用 LangChain structured_llm
  → LLM 返回 ExtractedEventList (Pydantic 对象)
  → 收集所有事件 → 去重合并
  → 对每个事件的 persons 列表：
      1. person 表按 name 精确匹配
      2. person_alias 表按 alias 精确匹配
      3. 都未找到 → 创建新 Person(status=draft) + PersonAlias
  → 创建 Event + PersonEvent 关联记录
  → 更新 event.persons JSONB 字段
```

**抽取字段**:

| 字段 | 类型 | 说明 |
|------|------|------|
| title | string | 事件标题，10-30 字 |
| description | string | 事件描述，1-3 句话 |
| start_date | string | ISO 8601 格式 |
| end_date | string | ISO 8601 格式 |
| display_time | string | 原文时间表述 |
| time_type | enum | POINT / PERIOD / FUZZY |
| granularity | enum | YEAR / MONTH / DAY / SEASON |
| event_type | enum | BIRTH / DEATH / EDUCATION / CAREER / CREATION / HISTORICAL / OTHER |
| location | dict | 地点信息 `{"name": "..."}` |
| **persons** | **list[string]** | **参与人物姓名列表（v2 新增）** |

**无时间信息处理**:
- 文本不含时间时仍提取事件
- `time_type` = FUZZY
- `start_date` / `end_date` = 关联人物的 `birth_date` 作为默认参照
- `display_time` = "时间不详"

**约束**:
- 抽取结果标记 `is_inferred=true`、`source='AI 抽取'`
- 同一文件中重复事件按 title + start_date 去重
- LLM 调用超时 60s/切片

**交互流程**:
1. 用户在文件管理页点击某行的"事件抽取"按钮
2. 按钮变为 loading 状态（disabled）
3. 后端后台执行抽取（status → extracting）
4. 前端轮询获取状态，完成后刷新列表
5. 抽取完成后 status 变为 completed

### 2.4 文件管理页面

**页面路径**: `/admin/upload`

**布局**:

```
┌──────────────────────────────────────────────────────┐
│  关联人物: [___搜索输入框___]      [上传文件按钮]      │
├──────────────────────────────────────────────────────┤
│  文件列表                                            │
│  ┌──────────────────────────────────────────────────┐│
│  │ 文件名 │ 大小 │ 类型 │ 人物 │ 状态 │ 时间 │ 操作  ││
│  ├──────────────────────────────────────────────────┤│
│  │ 三国志.pdf│1.2MB│PDF│诸葛亮│chunked│05-07│[抽取][切片][删除]│
│  │ 出师表.txt│12KB│TXT│诸葛亮│extracting│05-07│[抽取 ⟳][切片][删除]│
│  └──────────────────────────────────────────────────┘│
└──────────────────────────────────────────────────────┘
```

**状态标签颜色**:
- `uploaded` → blue（蓝色）
- `chunking` → orange（橙色）
- `chunked` → green（绿色）
- `extracting` → purple（紫色）+ loading 动画
- `completed` → success（成功绿）
- `error` → red（红色）

**操作按钮逻辑**:

| 按钮 | 可用条件 | 行为 |
|------|----------|------|
| 事件抽取 | status ∈ {chunked, completed} | 调用 POST /api/uploaded-files/{id}/extract |
| 事件抽取(禁用) | status = extracting | 灰色 + Loading 图标，不可点击 |
| 切片 | status ∈ {chunked, completed, extracting} | 跳转 /admin/upload/{fileId}/chunks |
| 删除 | 始终可用 | 确认弹窗后调用 DELETE /api/uploaded-files/{id} |

### 2.6 人物搜索选择器

**功能描述**: 可复用的 Ant Design Select 组件，支持模糊搜索人物姓名。

**行为**:
- 输入时防抖 300ms 后调用 `GET /api/persons?search=xxx`
- 下拉列表展示匹配的人物姓名
- 选中后返回 person_id
- 支持清空选择

## 4. 非功能需求

- **响应时间**: 上传 API < 2s，切片/抽取 API 异步执行
- **文件限制**: 最大 10MB，类型校验（仅 .txt/.pdf）
- **编码规范**: 每个函数必须写注释
- **提交规范**: Conventional Commits
- **设计原则**: 宁简勿繁，异步用 asyncio.create_task 而非 Celery
