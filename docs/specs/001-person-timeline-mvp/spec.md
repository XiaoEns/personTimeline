# 功能规格说明书 — 单人物时间轴 MVP

- **阶段编号**: 001
- **功能名称**: person-timeline-mvp
- **版本**: 1.0
- **状态**: 草稿

---

## 1. 概述

### 1.1 目标
实现单人物时间轴管理全流程：创建人物 → 手动/自动添加事件 → 时间轴可视化展示。确保用户可以在脱离 AI 能力的情况下完成所有核心操作。

### 1.2 范围

#### 包含功能
| 模块 | 功能 | 优先级 |
|------|------|--------|
| 人物管理 | 创建、编辑、删除、列表搜索 | P0 |
| 事件管理 | 创建、编辑、删除、列表搜索，支持点/段/模糊时间 | P0 |
| 人物-事件关联 | 绑定/解绑、角色标注、排序 | P0 |
| 别名管理 | 人物别名添加/删除 | P1 |
| 文件上传 | TXT/PDF 上传，提取文本并保存 | P1 |
| AI 抽取 | 调用大模型从文本中提取结构化事件 | P2 |
| 时间轴展示 | D3.js 单人物泳道图（点+段+模糊事件） | P0 |
| 事件详情 | 点击事件弹出详情卡片 | P0 |

#### 不包含功能
- 多人泳道对比（第二阶段）
- 事件对齐（第二阶段）
- OCR 图片文字识别（第三阶段）
- 权限认证（后续补充）
- 自动化测试（后续补充）
- Docker 部署（后续补充）

## 2. 功能规格

### 2.1 人物管理

**功能描述**: 用户可创建、编辑、删除人物，并通过列表页搜索和管理所有人物。

**字段说明**:
| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| name | string | 必填，最大 200 字符 | 人物姓名 |
| birth_date | datetime | 可选 | 规范化出生时间，精确到秒 |
| death_date | datetime | 可选 | 规范化逝世时间，精确到秒 |
| birth_display | string | 可选，最大 100 字符 | 原文出生表述，如"约1660年" |
| death_display | string | 可选，最大 100 字符 | 原文逝世表述 |
| avatar_url | string | 可选，最大 500 字符 | 头像图片链接 |
| summary | text | 可选 | 人物简介 |
| status | enum | 默认 draft | draft(草稿) / published(已发布) |

**交互流程**:
1. 进入人物列表页，默认显示所有人物
2. 可通过搜索框按姓名模糊搜索
3. 点击"新建"进入创建表单
4. 点击列表项进入编辑表单
5. 保存后返回列表页

### 2.2 事件管理

**功能描述**: 用户可创建、编辑、删除事件。事件支持三种时间类型：时间点（POINT）、时间段（PERIOD）、模糊时段（FUZZY）。

**字段说明**:
| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| title | string | 必填，最大 500 字符 | 事件标题 |
| description | text | 可选 | 事件描述 |
| start_date | datetime | 必填 | 事件开始时间 |
| end_date | datetime | 必填 | 事件结束时间（点事件与 start 相同） |
| display_time | string | 可选，最大 200 字符 | 原文时间表述 |
| time_type | enum | 必填 | POINT / PERIOD / FUZZY |
| granularity | enum | 必填 | YEAR / MONTH / DAY / SEASON |
| event_type | enum | 必填 | BIRTH / DEATH / EDUCATION / CAREER / CREATION / HISTORICAL / OTHER |
| location | json | 可选，默认 {} | 地点信息 |
| is_inferred | boolean | 默认 false | 是否 AI 推断 |
| source | string | 可选，最大 1000 字符 | 来源说明 |

**交互流程**:
1. 进入事件列表页，可按事件类型、关联人物、时间范围筛选
2. 点击"新建"进入创建表单
3. 创建时可选择关联人物并标注角色
4. 保存后返回列表页

### 2.3 人物-事件关联

**功能描述**: 事件可绑定到一个人或多个人物，每个人物在事件中可标注角色和个人视角描述。

**关联字段**:
| 字段 | 类型 | 说明 |
|------|------|------|
| role | string | 角色（主持/参与/见证/作者等） |
| personal_title | string | 该人物视角的原始事件标题 |
| personal_display_time | string | 该人物视角的时间原文表述 |
| is_primary | boolean | 是否主要人物 |
| sort_order | int | 排序权重 |

**交互流程**:
1. 在事件编辑页中，通过人物选择器搜索并选择人物
2. 标注该人物在此事件中的角色
3. 可在人物详情页查看其关联的所有事件

### 2.4 别名管理

**功能描述**: 为人物添加/删除别名（如字、号、曾用名），用于后续 NER 匹配。

**交互流程**:
1. 在人物编辑页中以标签形式展示别名
2. 输入别名后回车或点击添加
3. 点击标签上的删除按钮移除别名

### 2.5 文件上传与文本提取

**功能描述**: 用户可为人物上传传记文件（TXT/PDF），系统提取纯文本内容并保存。

**支持格式**:
- `.txt` — 直接读取
- `.pdf` — 使用 PyMuPDF 提取文本

**交互流程**:
1. 选择关联人物
2. 选择文件（支持拖拽）
3. 点击上传
4. 上传成功后显示文本预览（前 500 字符）

### 2.6 AI 事件抽取

**功能描述**: 调用大语言模型从传记文本中自动提取结构化事件。

**抽取内容**: 事件标题、时间表述、事件类型、地点、描述

**约束**:
- 抽取结果标记 `is_inferred=true`
- 抽取后自动创建 event 和 person_event 记录
- 用户需在事件列表中对 AI 事件进行确认或删除

**交互流程**:
1. 上传文本成功后，点击"AI 抽取"按钮
2. 系统同步调用大模型（连接超时 60s）
3. 展示抽取结果：提取出的事件列表
4. 用户可逐个确认（设置 is_inferred=false）或删除

### 2.7 时间轴展示

**功能描述**: 以 D3.js 单时间轴展示单人物的所有事件类型，以不同的颜色区分，提供沉浸式叙事阅读体验

**视觉要求**:
- 垂直布局，时间从上（旧）到下（新）
- 人物时间线：用一个线条展示，线条配置一个默认颜色，时间点事件（POINT）用实心圆点表示，颜色和线条颜色保持一致，时间段事件（PERIOD）用方形表示表示，模糊事件（FUZZY）在右侧以卡片形式展示
- 点状事件（POINT）：实心圆点，悬停显示浮层
- 时间段事件（PERIOD）：实心方形，悬停显示浮层，并在下方展示时间范围，时间范围也是一根时间线，时间的起始和结束要和时间轴上的时间刻度对齐
- 时间轴：放在底部，显示年份刻度，支持缩放和拖动
- 模糊事件（FUZZY）：在右侧以卡片的形式单独展示
- 出生/逝世：特殊图标标记在时间轴两端

**交互**:
- 悬停事件节点：显示标题和日期
- 点击事件节点：弹出详情卡片（EventCard）
- 鼠标滚轮：缩放时间范围

**数据格式**
- 时间需要去掉前置0，比如：0234 展示为 234

## 3. 数据模型

第一阶段使用以下 5 张表，详见 `docs/requirements/database-schema.md`：

| 表名 | 用途 | 第一阶段使用 |
|------|------|------------|
| person | 人物信息 | 全字段 |
| event | 标准事件 | 全字段 |
| person_event | 人物-事件关联 | 全字段 |
| person_alias | 人物别名 | 全字段 |
| biography_text | 原始文本 | 全字段 |

`external_person_info` 表在第三阶段使用。
